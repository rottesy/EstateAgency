1.3 Постановка задачи

Целью проекта является создание автоматизированной информационной системы управления агентством недвижимости на основе Qt6 и C++, обеспечивающей эффективное управление каталогом объектов недвижимости, ведение клиентской базы, контроль сделок и организацию аукционных торгов.

Основные задачи проекта:

1. Разработать модель данных для представления объектов недвижимости, клиентов, сделок и аукционов. Реализовать базовый класс Property и производные классы (Apartment, House, CommercialProperty) для различных типов недвижимости, класс Client для представления клиентов агентства, класс Transaction для сделок купли-продажи с различными статусами и классы Auction и Bid для организации аукционных торгов.

2. Реализовать систему управления объектами недвижимости. Обеспечить добавление, редактирование и удаление объектов различных типов, поиск объектов по идентификатору и адресу (город, улица, дом), отслеживание статуса доступности объектов и автоматическое обновление статуса при завершении сделок.

3. Реализовать систему управления клиентской базой. Обеспечить добавление новых клиентов с валидацией контактных данных (телефон, email), редактирование и удаление информации о клиентах, поиск клиентов по имени или номеру телефона, отслеживание даты добавления и истории взаимодействия с клиентами.

4. Реализовать систему управления сделками. Обеспечить создание сделок между клиентами и объектами недвижимости, управление статусами сделок (в ожидании, завершена, отменена), отслеживание финальной цены сделки и даты совершения, связывание сделок с объектами и клиентами, а также проверку доступности объектов для новых сделок.

5. Реализовать систему управления аукционными торгами. Разработать механизм создания аукционов для объектов недвижимости, управление ставками участников аукциона, отслеживание текущей максимальной ставки и истории ставок, автоматическое определение победителя аукциона, контроль статусов аукционов (активен, завершен, отменен) и поддержку функции выкупа (buyout).

6. Разработать графический пользовательский интерфейс. Создать главное окно приложения с навигацией по разделам системы, интерфейс для управления объектами недвижимости (список, добавление, редактирование, удаление, поиск), интерфейс для управления клиентами (список, добавление, редактирование, поиск), интерфейс для управления сделками (создание, просмотр, изменение статусов), интерфейс для управления аукционами (создание, просмотр ставок, управление торгами), отображение статистики и аналитики на дашборде, а также поиск данных во всех разделах.

7. Реализовать систему хранения данных. Обеспечить сохранение информации об объектах недвижимости, клиентах, сделках и аукционах в текстовые файлы, автоматическую загрузку данных при запуске приложения, использование текстового формата с разделителями для хранения данных и централизованное управление файлами через FileManager.

8. Обеспечить валидацию данных и обработку ошибок. Реализовать проверку корректности вводимых данных (цены, площади, адреса, телефонов, email, идентификаторов), обработку исключительных ситуаций на всех уровнях системы, информативные сообщения об ошибках для пользователя и предотвращение создания некорректных записей.

В результате выполнения проекта создана информационная система управления агентством недвижимости, обеспечивающая комплексное управление каталогом объектов различных типов, ведение клиентской базы, контроль сделок на всех этапах их жизненного цикла и организацию аукционных торгов. Система имеет интуитивный графический интерфейс, обеспечивает надежное хранение данных и может использоваться для автоматизации работы небольших и средних агентств недвижимости.
